# 유입 키워드 분석기 (Inflow Keyword Analyzer) — 설계 스펙 v1

> 작성일: 2026-03-07
> 목적: 네이버 블로그 유입 데이터에서 "검색 수요는 있는데 전용 글이 없는 키워드"를 자동으로 찾아내는 도구

---

## 1. 이 도구가 해결하는 문제

블로그에 사람들이 검색해서 들어오는 키워드 중에는 **전용 글이 없어서 이탈하는 키워드**가 있다.
예: "마운자로 투약 주기"로 매일 유입이 있는데, 이 주제에 대한 전용 글이 없음 → 글을 쓰면 확실한 트래픽 확보.

현재 이걸 알려면 매일 엑셀을 열어서 수동으로 확인해야 한다. 이 도구는 그 과정을 자동화한다.

---

## 2. 사용자 플로우 (매일 1분)

```
1. 네이버 블로그 관리자 → 유입분석 → 일간 → 엑셀 다운로드 (30초)
2. 대시보드(GitHub Pages) 접속
3. 엑셀 파일을 드래그앤드롭 (10초)
4. 즉시 분석 결과 확인:
   - "이 키워드로 글 써라" 리스트 (우선순위순)
   - 반복 등장 패턴 (3일 연속, 5일 연속 등)
   - 이미 커버된 키워드 vs 갭 키워드
5. 분석 결과가 자동으로 GitHub에 저장됨 (히스토리 누적)
```

## 4. 각 구성요소 상세

### 4-1. 대시보드 (docs/index.html) — 핵심

**GitHub Pages 정적 HTML. 서버 없음. 브라우저에서 모든 처리.**

#### 필요한 외부 라이브러리
- **SheetJS (xlsx.js)**: 브라우저에서 엑셀 파일 파싱
  - CDN: `https://cdn.sheetjs.com/xlsx-0.20.1/package/dist/xlsx.full.min.js`

#### 기능 1: 엑셀 업로드 + 파싱

드래그앤드롭 영역:
```
┌─────────────────────────────────────────────┐
│                                             │
│   📂 유입분석 엑셀 파일을 여기에 놓으세요    │
│   (여러 날짜 파일 한번에 가능)               │
│                                             │
└─────────────────────────────────────────────┘
```

**엑셀 파일 구조** (네이버 블로그 유입분석 → "전체" 다운로드):
```
Row 1: ('서비스명', '블로그 통계', None, None)
Row 2: ('데이터명', '유입분석', None, None)
Row 3: ('데이터 단위', '일간', None, None)
Row 4: ('데이터 기간', '2026.03.06. 일간', None, None)  ← 여기서 날짜 추출
Row 5: ('선택1', '전체', None, None)  ← "전체" 확인 (검색유입만 있는 파일도 처리 가능)
Row 6: ('다운로드 날짜', ...)
Row 7: (None, None, None, None)  ← 빈 행
Row 8: ('유입경로', '비율', '상세유입경로', '비율')  ← 헤더
Row 9~: 데이터 시작
```

**데이터 구조 (전체 선택 시)**:
```
('네이버 통합검색_모바일', '73.27', '마운자로 5mg 품절', '29.56')
(None, None, '마운자로 품절', '8.18')           ← 같은 유입경로의 하위 키워드
(None, None, '마운자로 입고', '2.83')
('네이버 블로그검색_모바일', '5.12', '마운자로 택배', '2.10')
```

**파싱 로직**:
1. Row 4에서 날짜 추출: "2026.03.06. 일간" → "2026-03-06"
2. Row 5에서 "전체" vs "검색 유입" 확인
3. Row 9부터 데이터 읽기:
   - 컬럼 A (유입경로)가 None이 아니면 → 새 유입경로 시작
   - 컬럼 C (상세유입경로) = 검색 키워드
   - 컬럼 D (비율) = 유입 비율 (%)
4. **검색 유입만 추출**: 유입경로가 "네이버 통합검색_모바일", "네이버 통합검색_PC", "네이버 블로그검색_모바일", "네이버 블로그검색_PC", "네이버 웹문서검색_모바일" 등 검색 관련인 것만 수집
5. URL 형태의 상세유입경로는 제외 (https://로 시작하는 것)
6. "기타" 항목은 제외

**출력 형식** (날짜별):
```json
{
  "2026-03-06": {
    "keywords": [
      {"keyword": "마운자로 5mg 품절", "ratio": 29.56, "source": "네이버 통합검색_모바일"},
      {"keyword": "마운자로 품절", "ratio": 8.18, "source": "네이버 통합검색_모바일"},
      {"keyword": "마운자로 입고", "ratio": 2.83, "source": "네이버 통합검색_모바일"}
    ]
  }
}
```

#### 기능 2: GitHub 연동 (히스토리 저장/로드)

**초기 설정** (최초 1회):
- GitHub PAT 입력란 (Health Trend Scanner와 동일한 패턴)
- PAT는 localStorage에 저장 (브라우저 닫아도 유지)

**대시보드 로드 시**:
1. GitHub API로 `data/inflow_history.json` fetch
2. GitHub API로 `data/my_posts.json` fetch
3. 과거 데이터를 메모리에 로드

**엑셀 업로드 후**:
1. 파싱된 오늘 데이터를 기존 히스토리에 머지
2. 이미 같은 날짜 데이터가 있으면 덮어쓰기 (중복 방지)
3. 머지된 inflow_history.json을 GitHub API로 커밋
4. 분석 결과 표시

#### 기능 3: 키워드 정규화

같은 의도의 검색어를 하나로 묶어야 함:

```
"마운자로 5mg 품절" = "마운자로5mg품절" = "마운자로 5mg 품절이유" = "마운자로 5미리 품절"
"마운자로 품절" = "마운자료 품절" = "마운자호 품절" = "마운자로품절" = "미운자로 품절"
```

**정규화 규칙** (우선순위순):
1. **공백 정규화**: 모든 공백 제거 후 비교
2. **오타 보정**: 편집거리(Levenshtein) 2 이하이면 같은 키워드로 간주
3. **포함 관계**: "마운자로 5mg 품절 이유"는 "마운자로 5mg 품절"의 하위 변형
4. **대표 키워드 선정**: 가장 높은 비율을 가진 형태를 대표로 사용

**정규화 결과**:
```json
{
  "group": "마운자로 5mg 품절",
  "variants": ["마운자로5mg품절", "마운자로 5미리 품절", "마운자로 5mg 품절이유"],
  "total_ratio": 35.2,
  "days_appeared": 6
}
```

#### 기능 4: 기존 글 매칭

`my_posts.json`에서 로드한 글 목록과 유입 키워드를 매칭:

**매칭 로직**:
1. 글 제목에 키워드의 핵심 단어가 포함되어 있으면 → "커버됨"
2. 글 본문(요약)에 키워드 핵심 단어가 2개 이상 포함 → "부분 커버"
3. 매칭 안 됨 → **"갭 키워드"** ← 이게 핵심 출력

**핵심 단어 추출**:
- 키워드에서 조사/접속사 제거: 은/는/이/가/을/를/의/에/에서/으로/과/와/도/만
- 예: "마운자로 투약 주기" → ["마운자로", "투약", "주기"]
- 글 제목 "마운자로 5mg 품절, 대처법 총정리" → ["마운자로", "5mg", "품절", "대처법", "총정리"]
- "마운자로 투약 주기"의 핵심단어 3개 중 1개만 매칭 → 갭 키워드

#### 기능 5: 갭 분석 + 우선순위

**최종 출력 — 글감 추천 리스트**:

```
🔴 즉시 작성 (5일+ 연속, 전용 글 없음)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1. "마운자로 투약 주기"
   📊 6일 연속 유입 | 평균 비율 0.63%
   📝 관련 글 없음
   💡 추천: 투약 간격, 늦었을 때 대처, 증량 스케줄 포함

2. "마운자로 택배"
   📊 4일 연속 유입 | 평균 비율 0.45%
   📝 관련 글 없음
   💡 추천: 택배 가능 여부, 콜드체인, 약국별 정책

🟠 작성 고려 (3일+ 연속, 전용 글 없음)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
3. "다이소 영양제"
   📊 3일 연속 유입 | 평균 비율 0.48%
   📝 관련 글 없음

🟡 모니터링 (2일 등장, 추이 관찰)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
4. "탈모약 건보"
   📊 2일 유입 | 평균 비율 0.62%
   📝 관련 글 있음 (부분 커버): "탈모약 건보적용, 약사가 정리하는 핵심 포인트"

✅ 이미 커버됨 (참고용)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
- "마운자로 5mg 품절" → 6일 연속 | 29.5% | 커버 글: "마운자로 5mg 품절..."
- "마운자로 품절" → 6일 연속 | 8.4% | 커버 글: "마운자로 5mg 품절..."
```

**우선순위 점수 공식**:
```
Priority Score = 연속등장일수 × 평균유입비율 × 갭배수

- 연속등장일수: 최근 며칠 연속 등장했는가 (1~30)
- 평균유입비율: 등장한 날들의 비율 평균 (0.1~50%)
- 갭배수:
  - 전용 글 없음 = ×3
  - 부분 커버 = ×1.5
  - 완전 커버 = ×0.3 (이미 있어도 보여주되 낮은 우선순위)
```

### 4-2. 글 목록 자동 업데이트 (scripts/update_posts.py)

**GitHub Actions로 하루 1회 실행.**

```python
# 의사 코드
def update_posts():
    # 1. RSS에서 글 목록 가져오기
    rss_url = f"https://rss.blog.naver.com/{BLOG_ID}.xml"
    rss_posts = parse_rss(rss_url)
    # 각 글: {title, url, date, summary}
    
    # 2. 기존 my_posts.json 로드
    existing = load_json("data/my_posts.json")
    
    # 3. 새 글만 추가 (URL 기준 중복 체크)
    for post in rss_posts:
        if post.url not in [p.url for p in existing]:
            # 4. 모바일 URL로 본문 가져오기 (선택적)
            try:
                body_text = scrape_mobile(post.url)
                post.body_keywords = extract_keywords(body_text)
            except:
                # 스크래핑 실패 시 RSS 요약만 사용
                post.body_keywords = extract_keywords(post.summary)
            
            existing.append(post)
    
    # 5. 저장
    save_json("data/my_posts.json", existing)
```

**모바일 스크래핑 상세**:
```python
def scrape_mobile(post_url):
    """
    네이버 블로그 모바일 버전에서 본문 텍스트 추출
    
    주의:
    - m.blog.naver.com 형태로 변환하여 요청
    - User-Agent를 모바일로 설정
    - 실패 시 None 반환 (RSS 요약으로 폴백)
    - 요청 간 2초 딜레이 (네이버 차단 방지)
    """
    mobile_url = convert_to_mobile(post_url)
    headers = {
        "User-Agent": "Mozilla/5.0 (iPhone; CPU iPhone OS 16_0 like Mac OS X) ..."
    }
    response = requests.get(mobile_url, headers=headers)
    
    if response.status_code != 200:
        return None
    
    soup = BeautifulSoup(response.text, 'html.parser')
    
    # 본문 영역 추출 (네이버 모바일 블로그 구조)
    # se-main-container 또는 postViewArea 클래스
    content = soup.select_one('.se-main-container') or soup.select_one('#postViewArea')
    
    if content:
        return content.get_text(separator=' ', strip=True)
    return None
```

### 4-3. GitHub Actions (daily_update.yml)

```yaml
name: Daily Post Update
on:
  schedule:
    - cron: '0 23 * * *'   # 매일 08:00 KST (UTC 23:00)
  workflow_dispatch:         # 수동 트리거 가능

jobs:
  update:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      - name: Setup Python
        uses: actions/setup-python@v5
        with:
          python-version: '3.11'
      
      - name: Install dependencies
        run: pip install requests beautifulsoup4 lxml
      
      - name: Update post list
        env:
          BLOG_ID: biopharmblog
        run: python scripts/update_posts.py
      
      - name: Commit changes
        run: |
          git config user.name "GitHub Actions"
          git config user.email "actions@github.com"
          git add data/my_posts.json
          git diff --staged --quiet || git commit -m "Auto: update post list $(date +%Y-%m-%d)"
          git push
```

## 6. 핵심 알고리즘 상세

### 6-1. 키워드 정규화 (JavaScript)

```
입력: ["마운자로 5mg 품절", "마운자로5mg품절", "마운자로 5미리 품절", 
       "마운자로 5mg 품절이유", "마운자로 5mg 품절 언제"]

Step 1: 공백/특수문자 제거 → 정규화된 형태로 변환
  "마운자로5mg품절", "마운자로5mg품절", "마운자로5미리품절",
  "마운자로5mg품절이유", "마운자로5mg품절언제"

Step 2: 포함 관계 체크
  "마운자로5mg품절" ⊂ "마운자로5mg품절이유" → 같은 그룹
  "마운자로5mg품절" ⊂ "마운자로5mg품절언제" → 같은 그룹

Step 3: 편집거리 (공백 제거 후)
  "마운자로5mg품절" vs "마운자로5미리품절" → 거리 2 → 같은 그룹

Step 4: 대표 키워드 = 가장 높은 비율을 가진 원본 형태
  → "마운자로 5mg 품절" (29.56%)

출력: {
  "representative": "마운자로 5mg 품절",
  "variants": ["마운자로5mg품절", "마운자로 5미리 품절", ...],
  "total_ratio_by_date": {"2026-03-06": 35.2, "2026-03-05": 34.1, ...}
}
```

### 6-2. 글 매칭 (JavaScript)

```
입력 키워드: "마운자로 투약 주기"
핵심 단어: ["마운자로", "투약", "주기"]

내 글 목록 순회:
  글 1: "마운자로 5mg 품절, 대처법 총정리"
    - 제목 단어: ["마운자로", "5mg", "품절", "대처법", "총정리"]
    - 매칭: "마운자로" 1개 → 3개 중 1개 = 33% → 매칭 부족
    - body_keywords: ["마운자로", "5mg", "품절", "대처법", "2.5mg", ...]
    - 본문 매칭: "마운자로" 1개 → 부분 커버 불충분
    → 결론: 갭 키워드 ✅

  글 2: "마운자로 용량별 투약 가이드" (가상)
    - 제목 단어: ["마운자로", "용량별", "투약", "가이드"]
    - 매칭: "마운자로", "투약" 2개 → 3개 중 2개 = 67% → 매칭!
    → 결론: 커버됨

매칭 기준:
  - 핵심단어의 60% 이상이 제목에 포함 → "완전 커버"
  - 핵심단어의 40% 이상이 제목+본문에 포함 → "부분 커버"  
  - 그 외 → "갭 키워드"
```

### 6-3. 우선순위 점수

```
Priority Score = consecutive_days × avg_ratio × gap_multiplier

예시:
  "마운자로 투약 주기": 6일 × 0.63% × 3(갭) = 11.34
  "마운자로 택배": 4일 × 0.45% × 3(갭) = 5.40
  "다이소 영양제": 3일 × 0.48% × 3(갭) = 4.32
  "마운자로 5mg 품절": 6일 × 29.5% × 0.3(커버됨) = 53.1 → 높지만 이미 글 있음
```

## 7. 엣지 케이스 처리

| 상황 | 처리 |
|:---|:---|
| 첫날 (히스토리 없음) | 1일치만으로 분석. 연속일수=1, 반복 패턴은 못 봄. "아직 데이터 부족" 안내 |
| 같은 날짜 엑셀 재업로드 | 기존 데이터 덮어쓰기 (최신 파일 우선) |
| "전체" 대신 "검색유입"만 선택한 엑셀 | 파싱 로직 다름 (컬럼 A=키워드, C=유입경로). Row 5의 선택1 값으로 분기 |
| 엑셀 파일이 아닌 파일 업로드 | 에러 메시지: "엑셀(.xlsx) 파일만 업로드 가능합니다" |
| my_posts.json 아직 없음 (Actions 미실행) | 갭 분석 없이 유입 키워드만 표시. "글 목록을 먼저 설정하세요" 안내 |
| GitHub PAT 미입력 | 히스토리 저장 안 됨. 분석은 가능하되 "PAT를 설정하면 히스토리가 저장됩니다" 안내 |
| RSS 피드 접근 실패 | Actions 로그에 경고. 기존 my_posts.json 유지 |
| 모바일 스크래핑 차단 | RSS 요약만으로 body_keywords 생성. 정확도는 낮아지지만 동작 |
| 30일 넘는 오래된 히스토리 | 자동 정리 없음 (파일 사이즈 문제 될 때까지 유지). 분석은 최근 30일만 사용 |

## 9. 기존 도구와의 관계

```
[유입 키워드 분석기] ← 이 도구
  질문: "내 독자가 뭘 원하는데 아직 안 줬나?"
  데이터: 내 블로그 유입 엑셀 (내부 데이터)
  갱신: 매일 (수동 업로드)
  
[Keyword Deep Dive]
  질문: "시장에서 약사가 쓸 만한 빈자리가 어디냐?"
  데이터: 네이버 검색/블로그 API (외부 데이터)
  갱신: 하루 4회 자동

[Health Trend Scanner]
  질문: "지금 당장 터지고 있는 게 뭐냐?"
  데이터: 네이버 뉴스/DataLab + AI (외부 데이터)
  갱신: 하루 4회 자동
```

세 도구는 독립적으로 동작하지만, 향후 연동 가능:
- Trend Scanner에서 발견한 키워드가 → 유입 분석기에서 실제 유입 확인되면 → "검증된 트렌드" 표시
- Deep Dive의 전문가갭 데이터를 → 유입 분석기의 우선순위 점수에 반영
